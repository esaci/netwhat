	- permet des echanges entre les ordi clients et le serveur

				IPV4 ; 4 chiffres entre 0 et 255   32 bits
				IPV6 : 8 chiffres entre 0 et 65535 128 bits

Modele TCP et UDP : 
- TCP mode connecte securie Transmission Control Protocol
	protocole le plus utilisé sur internet
	Les informations sont testées et vérifiés.
	si un ordinateur ne reçois pas correctement un packet, ce-dernier est renvoyé
	Le header de TCP est de 20 bytes

	TCP est utilisé pour les protocoles HTTP , HTTPS , FTP, SMTP

- UDP mode non connecte non fiable  User Datagram Protocol
	 Il n’y a pas de garantie sur la réception des données
	 ne necessite pas de verification a la reception
	 Le header de UDP est de 8 bytes

A quoi sert un serveur DHCP et qu’est-ce que le protocole DHCP ?

 - DHCP signifie Dynamic Host Configuration Protocol
 - un protocole qui permet à un ordinateur qui se connecte sur un réseau local d’obtenir dynamiquement et automatiquement sa configuration IP.

UDP :

Faux :
	UDP est fiable car il garantit la livraison des donnees au routeur de destination
Vrai :
	UDP NE DISPOSE AUE DU MECANISME DE CONTROLE DERREUR DE BASE UTILISANT DES SOMMES DE CONTROLE
	UDP EST UN PROTOCOLE ORIENTE DATAGRAMME
	UDP EST EN CHARGE DE LA DIFFUSION
	UDP EST PLUS RAPIDE ET EFFICACE AUE TCP
	IL BY A PAS DE SEQUENCEMENT DES DONNEE DANS UDP. SI LA COMMANDE EST REAUISE ELLE DOIT ETRE GEREE PAR LA COUCHE APPLICATIOn

TCP :

Faux :
	La livraison des donnees a la destination ne peut pas etre garantie en TCP
Vrai :
	TCP FOURNIT DES MECANISMES DE VERIFICATION DERRREUR ETENDUS. CEST PARCQUIL FOURNIT LE CONTROLE DE FLUX ET LAQUITTEMENT DES DONNEEES
	TCP EST COMPARATIVEMENT PLUS LENT AUE UDP
	TCP EST UN PROTOCOLE ORIENTE CONNEXION
	LE SEAUENCAGE DES DONNEES EST UNE CARACTERISTIAUE DE TCP ca arrive dans lordre
	TCP NE PREND PAS EN CHARGE LA DIFFUSION
